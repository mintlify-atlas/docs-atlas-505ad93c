The **Sequence Read Archive (SRA)** is NCBI's primary repository for raw sequencing data generated by next-generation sequencing platforms. It stores data from projects spanning genomics, transcriptomics, metagenomics, epigenomics, and more — making it one of the largest publicly available repositories of biological sequence data in the world.

The **SRA Toolkit** is a collection of command-line tools and libraries that lets you access, download, convert, and inspect data stored in the SRA. Whether you need to pull a single accession for a targeted analysis or integrate SRA data retrieval into a large-scale pipeline, the toolkit provides the utilities to do it efficiently.

## Key tools

<CardGroup cols={2}>
  <Card title="fasterq-dump" icon="bolt" href="/tools/fasterq-dump">
    Fast, multi-threaded extraction of reads from SRA accessions into FASTQ format. Replaces the older `fastq-dump` for most use cases and handles paired-end, single-end, and technical reads. Supports reads longer than 65K bases (since 3.3.0).
  </Card>
  <Card title="prefetch" icon="download" href="/tools/prefetch">
    Downloads SRA accessions and their dependencies to your local cache before conversion. Supports bulk downloads, cart files, AWS/GCP cloud access, and controlled-access dbGaP data via NGC files.
  </Card>
  <Card title="sra-pileup" icon="chart-bar" href="/tools/sra-pileup">
    Generates alignment pileup output and SAM format directly from aligned SRA (cSRA) data, useful for variant calling, coverage analysis, and integration with samtools.
  </Card>
  <Card title="vdb-dump" icon="database" href="/tools/vdb-dump">
    Inspects the raw contents of VDB (Virtual Database) objects. Useful for debugging, exploring column layouts, and examining data that higher-level tools may not expose.
  </Card>
  <Card title="vdb-config" icon="sliders" href="/tools/vdb-config">
    Manages toolkit configuration: cache locations, cloud credentials, proxy settings, and format preferences (SRA Normalized vs. SRA Lite). Supports both interactive TUI and command-line modes.
  </Card>
  <Card title="vdb-validate" icon="circle-check" href="/tools/vdb-validate">
    Validates the integrity of SRA data files by checking checksums, alignment ranges against reference lengths, and structural consistency.
  </Card>
  <Card title="sra-stat / sra-info" icon="chart-line" href="/tools/sra-stat">
    Retrieves metadata and statistics for SRA accessions — read counts, base counts, platform information, layout (paired/single), and fingerprinting data — without downloading the full dataset.
  </Card>
  <Card title="srapath" icon="route" href="/tools/srapath">
    Resolves the storage location of an SRA accession, returning the URL or file path where the data can be found. Useful for scripting and pipeline integration.
  </Card>
</CardGroup>

## Data formats

The toolkit works with several data formats:

- **SRA Normalized Format (`.sra`)** — the full-fidelity archive format with complete per-base quality scores.
- **SRA Lite (`.sralite`)** — a compressed variant with simplified quality scores (Q30 for pass, Q3 for reject). Significantly smaller than the normalized format; suitable for workflows that do not require precise quality values.
- **FASTQ** — the standard format for raw reads output by `fasterq-dump` and `fastq-dump`. Paired-end data is split into separate files (`_1.fastq`, `_2.fastq`) or combined depending on the flags used.
- **SAM/BAM** — alignment format output by `sra-pileup` for accessions that contain mapped reads.

## Supported platforms

The SRA Toolkit runs on:

- **Linux** (x86\_64 and arm64; built on AlmaLinux 8 as of release 3.3.0)
- **macOS** (x86\_64 and Apple Silicon)
- **Windows** (x86\_64)
